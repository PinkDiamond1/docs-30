Require Destination Tags

The RequireDest setting (requireDestinationTag in RippleAPI) is designed to prevent customers from sending payments to your address if they forgot the destination tag that identifies whom to credit for the payment. When enabled, the XAG Ledger rejects any payment to your address if it does not specify a destination tag.

The following is an example of using a locally-hosted rippled's submit method to send an AccountSet transaction to enable the RequireDest flag:

Request:

  • JSON-RPC
POST http://localhost:5005/
Content-Type: application/json

{
    "method": "submit",
    "params": [
        {
            "secret": "sn3nxiW7v8KXzPzAqzyHXbSSKNuN9",
            "tx_json": {
                "Account": "rf1BiGeXwwQoi8Z2ueFYTEXSwuJYfV2Jpn",
                "Fee": "15000",
                "Flags": 0,
                "SetFlag": 1,
                "TransactionType": "AccountSet"
            }
        }
    ]
}

Caution: Never submit a secret key to a server you do not control. Do not send a secret key unencrypted over the network.
